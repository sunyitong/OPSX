# Contributing

Contributions are welcome, especially:

- assembly notes,
- calibration logs,
- firmware fixes,
- schematic review,
- manufacturing feedback,
- Rust firmware development.

## Pull Requests

Please keep changes focused. For hardware changes, include enough context for review: affected nets, component references, manufacturing constraints, and any test results.

## Issues

When reporting hardware or firmware problems, include:

- board revision,
- fabrication settings,
- firmware version or commit,
- symptoms,
- photos or measurement notes when available.
